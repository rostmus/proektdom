Html разметка

Теги для разметки текста

  • p - параграф, абзац (paragraph)
  • h1-h6 - заголовки от большего к меньшему
  • blockquote - разметка блоков, цитат
  • q - цитата по тексту
  • strong сделать акцент на слове, фразе,обозначить важность
  • em - выделить слово
  • a - слово и фразу сделать гиперссылкой =href
  • img - вставка картинки в документ =src-ссылка =alt-обязательная подпись если не грузится
  • span> не имеет сысловой нагрзки - для выделения
  • figure> - для обозначения доски почета – выделения в рамку
  • figcaption> - для создания подписей к доске почета

Теги для списков

  • ul> - родительский тег – обозначение неупорядочного списка
  • ol> - родительский тег – обозначение упорядочного списка
  • li> - тег списка – перечисление наименований
  • dl> - список описаний
  • dt> - тег в который заключается термин

Теги для таблиц

  • dd> - тег в котором размещается определение таблицы
  • table> - начало таблицы + основа-дальше варианты
  • thead> - для обозначения заголовков – выделяется и жирным шрифтом
  • tbody> - для обозначения тела таблицы
  • tfoot> - для подсчета итогов
  • tr> - обозначение в таблице количества строк
  • td> - для остальных обозначений и перечисления ячеек

Теги для html верстки

  • html> - для обозначения вида документа =lang –для какого языка. Начинается и конч страницы
  • head>- для обозначения характеристик страницы. Не отображается на странице
  • body> - для обозначения тела страницы-все что показывается на сайте
  • title> - тот текст что показывается во вкладке
  • meta> - для обозначения кодировки =charset “utf-8”
  • link> - Для подгрузки библиотек или внешних ресурсов = href =rel-какой тип файла
  • !doctype html> - показывает браузеру по каким правилам отображать – в начало страницы всегда
  • script> - для подключении Java скриптов

Теги для разметки страницы

  • div>-не имеет смысловой нагрузки. используется для группирования элементов. присваиваем class= по смыслу,только на английском, строчные,не пробел а "-"
  • main>-используется для обозначения уникальной и основной части страницы
  • aside>-используется для обозначения части страницы, связанной по смыслу, но которую можно свободно удалить, не изменив смысл статьи и без ущерба для нее
  • section>-используется для тематически сгрупированного контента, у которого должен быть свой заголовок
  • article>-независимая область, можно вынести - смысл не изменится нигде, например рекламные баннеры, внутри должен быть свой заголовок
  • header>-для обозначения шапки сайта
  • footer>-для обозначения подвала сайта
  • nav>-для обозначения элементов навигации

Теги для вертски форм

  • input> позволяет создать поле для ввода текста и его отправки, имеет значения (type) text-текст, mail -почта, password-пароль
  • textarea> создание области в которую можно вводить текст в отличии от input где слово
  • button> - добавляет кнопку. имеет несколько хзначения. button-ничего не делает reset-сбрасывает значенияsubmint-кнопка отправки значения
  • form> вместо дива обьединяет все что связано в формой ввода - чтобы кнопки работали
  • label> - используется для ссылки акцента с наведения на текст input - в инпуту присвается id а лайбл используется for к id

Атрибуты для разметки текста

  • ="id"задать тегу универсальный индетификатор,позволяющий затем ссылаться на него
  • ="class"-задать один или несколько классов для управления внешнего вида
  • ="cite"-специальный атрибут для тега blockquote показывающий автора или источник
  • ="href" - сообщает браузеру куда перенаправлять пользователя в гиперссылке
  • ="title" - показывает подсказку при наведении на ссылку
  • ="src"указывает Url ссылку с изображением для тега img
  • ="alt" - показывает текстовое обьяснения картинки если оно не грузится

Атрибуты для списков и таблиц

  • ="start"атрибут в котором указывается с чего начать при разбивке списка
  • ="colspan" - обьединяет ячейки
  • ="rownspan" - объединяет столбцы

Атрибуты для верстки форм

  • ="placeholder" - атриут, для обозначении подсказки, тега input, что надо вводить
  • ="required" обьявляет поле для заполнения обязательным и проверяет перед отправкой форм
