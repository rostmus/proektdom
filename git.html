Свойства текста

  • Git init – инициализация каталога
  • Git add – за какими файлами следить
  • Git status – статус репозитория
  • Git commit – фиксация изменений с комментарием –a(все файлы после add) –m(сообщение) “название комита”
  • Git log – посмотреть историю коммитов “название” –graph –oneline – история конкретной ветки
  • Git show id> - полную инфу о коммите
  • Git rm – cashed название – удалить файл
  • Git revert id> - заменить последний коммит на новый
  • Git clone https: - клонирует удаленный репозиторий на комп
  • Git remote –v - управлять удаленным репозиторием add origin https – к лоакльному подключить удаленный
  • Git push – отправить изменения на сервер –u origin master – для первой отправки –tags- отправка тегов
  • Git branch – показать ветки “название” – создание ветки –d “название ветки” – удаление ветки с компа
  • Git checkout “имя ветки” – переключение на ветку --track создание локальной ветки с именем удаленной
  • Git push –u origin “название ветки” – заливает конкретную ветку
  • Git merge –nо –ff “название ветки” – слияние ветки
  • git tag – просмотр тегов –a v1.0 –m “название” –создание тега
  • git checkout комит> - переход к комиту любому
  • git reset - /hard –передвигать по комитам не созраняя /soft - -..-сохраняя в рабочем каталоге но не в индекс /mixed - и в индекс
  • git fetch origin> забирает изменения с удаленного сервера и сдвигают на нужную позицию
  • git pull сразу делает и fetch и merge
